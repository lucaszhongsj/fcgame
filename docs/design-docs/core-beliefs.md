# Core Beliefs（核心信念）

## 1. 事实先于结论

- 先读代码和运行行为，再下判断。
- 文档描述必须可追溯到代码事实。

## 2. 分层优先

- 交互逻辑、仿真逻辑、资源组织要分离。
- 避免跨层耦合和“临时补丁式”扩散。

## 3. 可维护性优先于短期速度

- 允许小步提交，不鼓励一次性大改。
- 每次改动都要留下可回顾的文档痕迹。

## 4. 一致性优先

- 输入映射、README、代码行为必须一致。
- 规范入口文档与子域文档保持链接完整。

## 5. 风险显式化

- 未完成能力（如 mapper 不完整）必须在文档中标注。
- 技术债不隐藏，统一进入 `tech-debt-tracker.md`。
